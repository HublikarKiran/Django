# CRUD Operations Notes

## What CRUD Means

CRUD means:

- Create
- Read
- Update
- Delete

In REST APIs, CRUD usually maps to HTTP methods:

```text
Create  -> POST
Read    -> GET
Update  -> PUT or PATCH
Delete  -> DELETE
```

## Where CRUD Is Configured In This Project

Main file:

```text
studentsystem/api/views.py
```

CRUD is provided through:

```python
viewsets.ModelViewSet
```

Example:

```python
class SubjectViewSet(viewsets.ModelViewSet):
    queryset = Subject.objects.all().order_by('name')
    serializer_class = SubjectSerializer
    permission_classes = [AdminWritePermission]
```

`ModelViewSet` automatically gives:

- `list()` for `GET /api/subjects/`
- `create()` for `POST /api/subjects/`
- `retrieve()` for `GET /api/subjects/{id}/`
- `update()` for `PUT /api/subjects/{id}/`
- `partial_update()` for `PATCH /api/subjects/{id}/`
- `destroy()` for `DELETE /api/subjects/{id}/`

## Where CRUD Routes Are Generated

Main file:

```text
studentsystem/api/urls.py
```

Example:

```python
router = DefaultRouter()
router.register('subjects', views.SubjectViewSet)
```

This creates all CRUD URLs for subjects.

## Create Operation

HTTP method:

```text
POST
```

Example:

```text
POST /api/subjects/
```

Example request:

```json
{
  "name": "Django",
  "description": "Django REST Framework"
}
```

In this project, some create operations override `perform_create()`.

Example:

```python
def perform_create(self, serializer):
    serializer.save(created_by=self.request.user)
```

This means the logged-in user is automatically saved as the creator.

## Read Operation

List all records:

```text
GET /api/subjects/
```

Read one record:

```text
GET /api/subjects/1/
```

Read access depends on permission classes.

For many endpoints, this project uses:

```python
AdminWritePermission
```

That means authenticated users can read, but only admins can write.

## Update Operation

Full update:

```text
PUT /api/subjects/1/
```

Partial update:

```text
PATCH /api/subjects/1/
```

Use `PUT` when sending all fields. Use `PATCH` when sending only changed fields.

Example:

```json
{
  "name": "Advanced Django"
}
```

## Delete Operation

HTTP method:

```text
DELETE
```

Example:

```text
DELETE /api/subjects/1/
```

Delete access depends on the endpoint permission.

## Special CRUD Behavior In This Project

### UserViewSet

Permission:

```python
permission_classes = [AdminOnlyPermission]
```

Only admins can access user CRUD.

### AssignmentSubmissionViewSet

Normal users see only their own submissions:

```python
return queryset.filter(student=self.request.user)
```

When creating a submission, the student is set automatically:

```python
serializer.save(student=self.request.user)
```

### ChatMessageViewSet

Normal users see only their own chat messages:

```python
return ChatMessage.objects.filter(user=self.request.user)
```

When creating a chat message, the answer is generated by the chatbot:

```python
answer=build_study_answer(question)
```

### NotificationViewSet

The creator is set automatically:

```python
serializer.save(created_by=self.request.user)
```

## Common API Calls

List:

```bash
curl -u username:password http://127.0.0.1:8000/api/subjects/
```

Create:

```bash
curl -X POST http://127.0.0.1:8000/api/subjects/ \
  -u admin:password \
  -H "Content-Type: application/json" \
  -d "{\"name\":\"Django\"}"
```

Update:

```bash
curl -X PATCH http://127.0.0.1:8000/api/subjects/1/ \
  -u admin:password \
  -H "Content-Type: application/json" \
  -d "{\"name\":\"DRF\"}"
```

Delete:

```bash
curl -X DELETE http://127.0.0.1:8000/api/subjects/1/ \
  -u admin:password
```
