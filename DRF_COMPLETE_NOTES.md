# Django REST Framework Notes

## What DRF Is

Django REST Framework, usually called DRF, is a toolkit for building APIs in Django. A normal Django view usually returns HTML. A DRF API view usually returns structured data, most commonly JSON.

DRF is useful because it gives Django:

- serializers, which convert models to JSON and validate incoming JSON
- viewsets, which provide list, create, retrieve, update, and delete actions
- routers, which automatically generate API URLs
- authentication classes, which identify the user
- permission classes, which decide what the user can do
- browsable API pages in the browser during development

## Where DRF Is Enabled In Django Settings

Configured file:

```text
studentsystem/studentsystem/settings.py
```

DRF is added inside `INSTALLED_APPS`:

```python
INSTALLED_APPS = [
    ...
    'rest_framework',
    ...
]
```

This makes DRF available to the Django project.

## Global DRF Configuration In This Project

Configured in:

```text
studentsystem/studentsystem/settings.py
```

Current configuration:

```python
REST_FRAMEWORK = {
    'DEFAULT_AUTHENTICATION_CLASSES': [
        'rest_framework.authentication.SessionAuthentication',
        'rest_framework.authentication.BasicAuthentication',
    ],
    'DEFAULT_PERMISSION_CLASSES': [
        'rest_framework.permissions.IsAuthenticated',
    ],
}
```

Meaning:

- `SessionAuthentication`: API requests can use the same logged-in browser session as the web app.
- `BasicAuthentication`: API requests can use username and password through HTTP Basic Auth.
- `IsAuthenticated`: by default, API endpoints require a logged-in/authenticated user.

## Health API

Defined in:

```text
studentsystem/api/views.py
```

Code:

```python
@api_view(['GET'])
@permission_classes([permissions.AllowAny])
def health(request):
    return Response({'status': 'ok'})
```

Route:

```text
GET /api/health/
```

This endpoint is public because it uses:

```python
permissions.AllowAny
```

## API Request Examples

Health check:

```bash
curl http://127.0.0.1:8000/api/health/
```

Authenticated request using Basic Auth:

```bash
curl -u username:password http://127.0.0.1:8000/api/subjects/
```

Create a subject:

```bash
curl -X POST http://127.0.0.1:8000/api/subjects/ \
  -u admin:password \
  -H "Content-Type: application/json" \
  -d "{\"name\":\"Python\",\"description\":\"Python subject\"}"
```

Patch a subject:

```bash
curl -X PATCH http://127.0.0.1:8000/api/subjects/1/ \
  -u admin:password \
  -H "Content-Type: application/json" \
  -d "{\"name\":\"Advanced Python\"}"
```

Delete a subject:

```bash
curl -X DELETE http://127.0.0.1:8000/api/subjects/1/ \
  -u admin:password
```

## Important Project-Specific Behavior

In `studentsystem/api/views.py`:

- `AdminOnlyPermission` protects user management APIs.
- `AdminWritePermission` lets authenticated users read but only admins write.
- `AssignmentSubmissionViewSet` filters normal users to their own submissions.
- `ChatMessageViewSet` filters normal users to their own chat messages.
- `ChatMessageViewSet.perform_create()` calls the chatbot answer generator automatically.
